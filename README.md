## 部署方式（任选其一）

### 方式一：GitHub Pages（推荐）
1. 在 GitHub 新建一个公开仓库（如 `baoji-prototype`）
2. 将 `deploy` 文件夹内的三个文件上传到仓库根目录
3. 进入仓库 Settings → Pages → Source 选择 `main` 分支
4. 访问 `https://你的用户名.github.io/仓库名/` 即可在线演示

### 方式二：Netlify（拖拽部署）
1. 打开 https://app.netlify.com/drop
2. 将 `deploy` 文件夹直接拖拽到网页中
3. 立即获得在线访问链接

### 方式三：Vercel（拖拽部署）
1. 打开 https://vercel.com/new
2. 选择 Import Git Repository 或拖拽文件夹
3. 一键部署并获取在线链接

### 方式四：本地直接打开
直接用浏览器打开 `index.html` 文件即可本地演示，无需服务器。

## 原型功能清单

本原型包含以下核心模块的交互页面：

**Web端管理后台：**
- 首页仪表盘（数据看板、客流趋势、待办事项）
- 前厅管理（客流识别、定人定岗、服务效率）
- 后厨管理（出品监控、合规检测）
- 供应链管理（库存管理、采购建议）
- 巡检管理（闭环管理、整改流程）
- 运营管理（培训管理）
- 系统管理（权限管理）

**移动端小程序：**
- 首页（数据概览、快捷入口、告警列表）
- 前厅（客流状态、翻台监测、座位可视化）
- 后厨（出品评分、合规检测、安全监控）
- 库存（库存查询、采购建议、效期预警）
- 巡检（自检任务、整改闭环）
- 我的（消息中心、培训学习、绩效查看）

## 技术说明

- 纯静态 HTML，无需后端服务器
- 内联 CSS + JavaScript，无外部依赖
- 使用 SVG 自绘图表，无需图表库
- 支持角色切换（管理员/店长/店员/督导）
- 响应式布局，适配 PC 和手机

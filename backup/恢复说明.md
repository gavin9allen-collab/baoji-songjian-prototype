## 恢复方式（任选一种）

### 方式一：本地文件直接恢复（最快）

如果 GitHub 仓库更新后出现问题，直接用本地备份覆盖：

```bash
# 1. 进入项目目录
cd deploy

# 2. 用备份文件覆盖当前文件（注意替换时间戳）
cp backup/web_backup_20260630_111937.html web.html
cp backup/mobile_backup_20260630_111937.html mobile.html
cp backup/index_backup_20260630_111937.html index.html

# 3. 重新推送到 GitHub
git add web.html mobile.html index.html
git commit -m "Revert to stable backup version"
git -c http.sslVerify=false push origin main
```

### 方式二：使用 Git 分支恢复（推荐）

如果 `backup-stable` 分支已推送到 GitHub：

```bash
# 将备份分支强制覆盖 main 分支
cd deploy
git checkout backup-stable
git branch -f main backup-stable
git checkout main
git -c http.sslVerify=false push -f origin main
```

### 方式三：使用 Git Tag 恢复

如果 `v1.0-stable` Tag 已推送到 GitHub：

```bash
cd deploy
git checkout v1.0-stable
git branch -f main v1.0-stable
git checkout main
git -c http.sslVerify=false push -f origin main
```

### 方式四：GitHub 网页端恢复（无需命令行）

1. 打开 GitHub 仓库页面：https://github.com/gavin9allen-collab/baoji-songjian-prototype
2. 点击右上角 **Branch** 下拉菜单，切换到 `backup-stable` 分支
3. 确认该分支文件内容正确
4. 进入 **Settings → Pages**，确认 Pages 源分支为 `main`
5. 将 `backup-stable` 的代码复制粘贴回 `main` 分支，或使用 Pull Request 合并

## 更新备份的操作流程

每次完成一轮修改并确认功能正常后，执行以下操作更新备份：

```bash
cd deploy

# 1. 本地文件备份
cp web.html backup/web_backup_$(date +%Y%m%d_%H%M%S).html
cp mobile.html backup/mobile_backup_$(date +%Y%m%d_%H%M%S).html
cp index.html backup/index_backup_$(date +%Y%m%d_%H%M%S).html

# 2. 更新 Git 备份分支
git branch -f backup-stable main
git -c http.sslVerify=false push -f origin backup-stable

# 3. 更新 Git Tag（可选，删除旧 tag 再重建）
git tag -d v1.0-stable 2>/dev/null
git tag v1.0-stable
git -c http.sslVerify=false push origin v1.0-stable --force
```

## 常见问题

### Q: 为什么 `git push` 要加 `-c http.sslVerify=false`？

A: 当前网络环境连接 GitHub 有 TLS 证书验证问题，加此参数可绕过证书验证完成推送。

### Q: 备份分支推送失败了怎么办？

A: 先使用**本地文件备份**恢复，等网络正常后再推送分支。本地备份文件 (`deploy/backup/*.html`) 是最可靠的恢复源。

### Q: 可以删除旧备份文件吗？

A: 建议保留最近 3~5 个版本的备份。如果磁盘空间充足，可以保留更多历史版本便于回溯。

### Q: 如何确认恢复后的文件是否正确？

A: 恢复后建议用以下方式验证：
1. 在浏览器中打开 `web.html`，确认页面正常显示
2. 按 `F12` 打开开发者工具 → Console 标签，确认没有红色报错
3. 测试登录按钮、页面切换等核心功能
